GHOSTS Data Products

The GHOSTS survey paper details the analysis and reduction of the survey data. Part of that work is summarized below in describing the data available from this website.

The Stellar Catalog - *.stars.fits.gz

This FITS file contains the final filtered catalog of stars, listed in separate binary tables for photometry using either the sparse- or crowded-field parameters together with the corresponding selection criteria described in Section 4.2 of the GHOSTS survey paper. The columns are:

  1. X - relative to the F814W drizzled image
  2. Y - relative to the F814W drizzled image
  3. RA - computed from the F814W drizzled image
  4. DEC - computed from the F814W drizzled image
  5. MAG1_ACS - Magnitude in the filter listed under the header keyword "FILTER1" (typically F606W) in VEGAMAG
  6. MAG1_JC - Magnitude in the equivalent Johnson-Cousins filter (V for F606W), computed using the updated Sirianni et al. (2005) transformations
  7. MAG1_ERR - Magnitude error in VEGAMAG
  8. CHI1 - Chi value of the fit for the "FILTER1" detection
  9. SNR1 - Signal to Noise ratio for the "FILTER1" detection
  10. SHARP1 - Sharpness of the "FILTER1" detection (see DOLPHOT Manual)
  11. ROUND1 - Roundness of the "FILTER1" detection (see DOLPHOT Manual)
  12. CROWD1 - Crowding of the "FILTER1" detection (see DOLPHOT Manual)
  13. FLAG1 - Error flag for the "FILTER1" detection. An error flag of zero means the star was recovered extremely well in the image, other values added to the flag indicate the type of problem: (+1) photometry aperture extends off chip, (+2) too many bad or saturated pixels, (+4) saturated at center of star, (+8) extreme case of one of the above
  14. MAG2_ACS - Magnitude in the filter listed under the header keyword "FILTER2" (typically F814W) in VEGAMAG
  15. MAG2_JC - Magnitude in the equivalent Johnson-Cousins filter (I for F814W), computed using the updated Sirianni et al. (2005) transformations
  16. MAG2_ERR - Magnitude error in VEGAMAG
  17. CHI2 - Chi value of the fit for the "FILTER2" detection
  18. SNR2 - Signal to Noise ratio for the "FILTER2" detection
  19. SHARP2 - Sharpness of the "FILTER2" detection (see DOLPHOT Manual)
  20. ROUND2 - Roundness of the "FILTER1" detection (see DOLPHOT Manual)
  21. CROWD2 - Crowding of the "FILTER1" detection (see DOLPHOT Manual)
  22. FLAG2 - Error flag for the "FILTER2" detection. An error flag of zero means the star was recovered extremely well in the image, other values added to the flag indicate the type of problem: (+1) photometry aperture extends off chip, (+2) too many bad or saturated pixels, (+4) saturated at center of star, (+8) extreme case of one of the above

The header of the FITS file also lists the zero-point corrections applied, together with additional data such as completeness at F606W-F814W = 0.75 (as computed from the artificial star tests) and the detected TRGB magnitude.

Raw Photometry - *.phot_sparse.gz *.phot_crowded.gz

These files contain the raw *.phot output from the DOLPHOT photometry code. The columns in the files are described in the DOLPHOT Manual.

DOLPHOT Parameter Files - *.param

These files list the input parameters for the DOLPHOT photometry code. Further details can be found in the DOLPHOT Manual.

Artificial Star Tests - *.artificial.fits.gz

This FITS file contains the three-dimensional probability distribution of recovering a star as a function of the F606W-F814W color, magnitude in the F606W filter and background sky brightness (measured from the Sky image). Bins without sufficient coverage in the artificial star tests to compute a probability are set to zero. The axes ranges are listed in the header of each three-dimensional image, with the exception of sky brightness, which is given separately as a binary table embedded in the same fits file, and lists the sky brightness of each bin.

- Note that as of November, 2011, the artificial star tests are not yet available for most of the "Crowded" field runs.

Multidrizzling - *.drz.fits.gz

The cleaned reference images in the F606W and F814W filters as computed with MultiDrizzle. We followed the standard steps outlined in the MultiDrizzle Handbook (Fruchter & Sosey 2009) but initially ran TweakShifts on all the CCD images for a given target. This task computed any residual shifts between the WCS of the observations and ensured that the final drizzled images for both filters used the same coordinate system. As MultiDrizzle updates the header of the *_flt files, we also turned off sky subtraction, and correspondingly set pixfrac = 1.0, to ensure proper treatment of Poissonian noise by the photometry code. The image from the F814W filter is used for coordinate references.

Galaxy Mask - *.gmsk.fits.gz

Where applicable, we provide the galaxy mask that was used to exclude spurious detections of extended objects coincident with the host galaxy disk, and to exclude high extinction regions when calculating the TRGB distance. These are computed by clipping the sky image at a given brightness limit before sub-sampling with a 15x15 IRAF blkav kernel, smoothing with a 25x25 IRAF median filter, and block replicating back to the original size with IRAF blkrep.

Data Mask - *.seg.fits.gz

This fits file contains the segmentation mask, which identifies the extended objects from the Source Extractor catalog in the drizzled images. The mask is passed through a maximum filter (7 x 7 pixels in size) to pad the mask for each extended object. This padding ensures that detections which lie on the periphery of an extended object are removed, as typically these detections are associated with the extended object itself.

During the initial photometry stages, it was observed that the photometry code was less effective at detecting sources within the halos of bright foreground stars. In fields far from the main galaxy disk, this effect can be corrected by artificial star tests. However, this is not necessarily the case in fields where bright regions are not only due to the halos of these foreground stars but also due to the galaxy disk itself. To ensure this effect did not impinge upon our results, we excluded by hand a larger region surrounding these few foreground stars. These regions are included in the segmentation mask to ensure proper calculation of the survey window.

SourceExtractor Catalog - *.se

the F814W SExtractor catalog from which the segmentation mask is constructed. The SExtractor parameters used are described in the GHOSTS survey paper.

The SExtractor catalog contains structural parameters that have proven useful to separate galaxies from stars in previous work (e.g., Holwerda et al. 2005). These parameters include a simple concentration index, defined as the ratio of the peak surface brightness to the total luminosity in the pixels above 1.6 sigma (mu_max/mag_iso), the number of pixels in the object (isoarea_image), the ellipticity (= 1-minor/major axis ratio), and the FWHM (fwhm_image). SExtractor's own star/galaxy classification is also included even though the procedure has not been calibrated on HST images and becomes random well above the detection limit.

Sky Image - *.sky.fits.gz

The background sky image computed by running the calcsky routine included in the DOLPHOT package on the drizzled F814W image. This file is used for sky brightness in the artificial star tests.

TRGB Plots

Where available, a plot of the tip of the red giant branch (TRGB) detection is shown for an individual field.

To measure the apparent magnitude of the TRGB, we follow the same method employed by Seth et al. (2005a). Specifically, we used the logarithmic edge-detection algorithm defined in Mendez et al. (2002) on stars selected with a color cut to remove both main sequence and redder asymptotic giant branch stars. This cut was typically set at 0.3 < F606W - F814W < 1.6 but for fields with significant helium-burning sequences the range was shifted to redder colors (>1.0 mag). These intermediate-age stars are typically confined to the galaxy disks, but for a few systems, notably NGC 2403 and NGC 5236, they can be far removed from the galaxy center. Generally, we only report TRGB measurements for fields that do not cover the main galaxy disk. This further minimizes contamination from younger stars and reduces the effects of internal extinction and crowding. However, for systems with only one field, or for galaxies that are sufficiently distant to easily discriminate the disk and halo populations in the same field, we still measured the TRGB. For these fields, we excluded the stars in regions of high surface brightness by using the galaxy masks described above. Additionally, we only included fields with enough stars for convergence in the detection. Typically, this required at least 10 stars within the color cut and within 0.25 mag of the TRGB.

The upper panel of these plots show the number of stars in the culled region as a function of F814W magnitude, while the middle panel plots the response of the Mendez et al. (2002) edge-detection algorithm to this distribution.

Random errors in the TRGB detections were assessed by reprocessing 500 Monte Carlo resamples of the stellar magnitudes with replacement. At the TRGB magnitude, these random errors dominate the stellar photometric errors reported by DOLPHOT. Hence an uncertainty in the TRGB magnitude was assigned by fitting a Gaussian to the distribution of the resamples. This fits, as well as the resamples are plotted in the bottom panel.
